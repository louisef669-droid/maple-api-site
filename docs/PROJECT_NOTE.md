# 개발 규칙

항상 현재 코드 기준으로 수정한다.

수정 전 파일명을 먼저 표시한다.

작은 단위로 수정한다.

기능 완료 후

git status

git add .

git commit

git push

# ChatGPT 개발 규칙

현재 프로젝트 기준으로 설명한다.

오래된 코드 기준으로 답변하지 않는다.

파일명 먼저 표시한다.

필요한 부분만 수정한다.

통째 교체는 필요한 경우에만 한다.

대화가 길어져 컨텍스트가 꼬일 가능성이 있으면
새 채팅을 먼저 권장한다.
